# Project Context: Personal Website Repository

## 1. Project Overview
*   **Goal:** To host and manage a personal website, showcasing projects, documentation, and personal information. This repository serves as the source for the `bradford1040.github.io` website.
*   **Environment:** Development and deployment are managed from a CachyOS (Arch-based) environment, with a focus on performance and bare-metal efficiency.

## 2. Key Areas
*   **Website Content:** Management of HTML, CSS, JavaScript, and Markdown files for the website's structure, styling, and content.
*   **Documentation:** Hosting and maintaining project documentation and personal notes in Markdown format.
*   **Version Control:** Utilizing GitHub for repository management, deployment, and collaboration.

## 3. Code Style & Mindset
*   **Tone & Depth:** Skip entry-level explanations or generic "how-to" filler. Provide highly specific, advanced technical data. 
*   **Engineering approach:** Solutions should respect a professional background in custom heavy-duty welding, structural fabrication, and master carpentry—prefer heavy, built-to-last designs over quick fixes.
*   **Indentation:** Use 2-space indentation for all code and configurations.
*   **Languages:** All JavaScript must use modern ES6 syntax. Follow PEP 8 rules for Python. Prioritize pure `bash` for system automation.

## 4. Conventions & Stack
*   **Package Managers:** Use `pacman` or `yay` exclusively for CachyOS/Arch repositories.
*   **Preferred Tools:** 
*   *Text/Code Editor:* Kate (leveraging custom External Tools, terminal pipelines, and custom keybindings) or VS Code Insiders for GUI environments; Nano for terminal-based editing.
*   *File Manager:* Krusader (dual-pane efficiency).
*   *Terminal:* Yakuake (drop-down CLI access) running Fish shell.
*   **Version Control:** Heavy reliance on GitHub CLI (`gh`) for seamless, terminal-driven repository management and Gist snippet deployments.
*   **Documentation:** Maintain all engineering blueprints, software logs, and project notes in clean, scannable Markdown (`.md`).
